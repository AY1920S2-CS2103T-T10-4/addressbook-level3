PROJECT: Nice and Amazing Student Assistant (NASA)

Overview

Nice & Amazing Student Assistant (NASA) is an application for busy university students to manage their module assignments, lessons, exams and consultations etc.

Summary of contributions

  • Major enhancement: Conceptualised and implemented unique GUI for NASA.

    • What it does: Allows users to view NASA data in an organized manner by displaying modules in columns.

    • Justification: This feature allows users easily view all of their activities which allows them to get a better gauge the number of activities.

    • Highlights: GUI is unique and differs greatly from traditional todo applications. There are many challenges involved in this enhancement:

      • Configuring the layout and display of the module tab alone required much planning due to the in depth nature of our data and display.

      • As our data was layered in that both modules and activities can be updated, it was a challenge to produce a responsive UI which could be updated along with data.

      • Additionally, typical GUI requirements such as resizing along with the window size and positioning had to be taken into consideration.

  • Major enhancement: added hint popup for commands with ability to copy example

    • What it does: detects which command the user wishes to use and provides a hint to show command usage. Additionally, a copy example message is provided so users can quickly utilise example messages and edit them as required.

    • Justification: This feature improves user experience as they do not have to repeatedly refer to the user guide to check what parameters to include. Through the pop up, the users are informed of what to include in the command, reducing the change of error and thus reducing the frustration of wrong input and error messages. Furthermore, allowing the user to copy the example reduces the need to check for the right prefixes included.

    • Highlights: This enhancement affects existing commands and commands to be added in future. The implementation too was challenging as it required configuring the GUI.

  • Minor enhancement: increase CLI-friendliness by adding keyboard shortcuts for user to navigate NASA more easily. Users can look for previous commands using up/down keys similar to the functionality in bash. In addition, they can easily navigate from tab to tab either using tab (in one key) or using the view command.

  • Minor enhancement: added "statistics display" to provide users with visualisation on the proportion of deadlines for all modules and the total tasks completed and not completed for each.

  • Minor enhancement: responsible for storage of data.

Hints

Contributed by: CharmaineKoh

New to NASA? Not to worry! NASA’s responsive hint pop up is here to guide you through all the commands.

Simply key in any valid command such as deadline and a pop up will appear informing you on exactly which parameters to include.

In addition, you can easily copy the example to your keyboard using the button on the popup. Now you can paste the command and view the result easily.

Command Format

  • Words in UPPER_CASE are the parameters to be supplied by the user e.g. in add m/MODULE_CODE, MODULE_CODE is a parameter which can be used as add m/CS1231.

  • Items in square brackets are optional fields e.g [n/Notes].

  • Items with …​ after them can be used multiple times including zero times e.g. [m/MODULE_CODE]…​ means m/CS2103T CS2101 …​..

  • Parameters can be in any order.

  • For items without …​, should multiple values for the same field be provided e.g m/CS2101 m/CS2103T, only the last value will be used, unless stated otherwise.

  • Commands are case-insensitive, unless stated otherwise.

  • Prefixes are case-sensitive e.g m/ is valid while M/ is invalid.

=== Saving the data Contributed by: CharmaineKoh

NASA data is saved by default in nasabook.json in the hard disk automatically after any command that changes the data. There is no need to save manually.

You can edit the data file directly to change any data.

If there is any invalid input in the data file, NASA will override the file and start with an empty file.

==== Implementation Event listener is added to command box and detects changes key presses. For each key press, NASA checks if the ctext starts with a valid command and updates the popup accordingly.

==== Internal workings for the popup

Below is an example of a user typing in the command "deadline" by for the week and how the underlying system works.

  1. The user starts to type in `deadline' into command box

  2. NASA checks as each character is typed if the string forms a valid command

  3. Once the command deadline is fully typed in, CommandBox calls the MainWindow.

  4. MainWindow sends command to be displayed to HintWindow.

  5. HintWindow retrieves hint from CommandHint to be displayed on hint window.

  6. HintWindow displays to show user parameters of deadline and its example.
